Modal Revit / pyRevit Mini IDE

PyPiPie Shell Help

This help page describes the current modal PyPiPie Shell for pyRevit / Revit. It covers editor behavior, buttons, shortcuts, file actions, run behavior, and status colors.

Because the tool is modal, it stays stable inside Revit and avoids the modeless focus and event issues that were causing trouble earlier.

What it is

A simple Python editor and execution window for Revit/pyRevit, built as a modal WPF dialog. The editor uses a multiline text box with line numbers and an output pane.

What it exposes

Your run scope includes doc, uidoc, uiapp, app, revit, sys, clr, and from Autodesk.Revit.DB import *.

Why modal matters

The shell opens with a blocking dialog. That keeps input, focus, and direct execution predictable inside Revit.

Window Layout

  • Top toolbar: file, search, and editing commands.
  • Editor: multiline code input with line numbers.
  • Splitter: drag to resize editor vs output.
  • Output pane: standard output and error / traceback display.
  • Status bar: file state, caret position, and color feedback.
  • Bottom row: Clear Output, Run, and Close.

File Operations

The editor works with normal .py files. Opening a file replaces the current editor contents. Save writes the entire editor buffer to disk.

When unsaved changes exist and you attempt to start a new file, open a different file, or close the window, a custom large-text dialog asks whether to save changes first.

Unsaved changes prompt: Yes = save first, No = discard changes, Cancel = stay in the editor.

Running Code

When you run code, the editor contents are normalized to \n line endings and then executed with exec(...) inside the current scope.

  • The output pane inserts a separator line: --- Run ---
  • Standard output is shown in black.
  • Tracebacks and errors are shown in red.
  • Successful runs end with [done].
  • Executed buffers are stored in run history for AltUp / AltDown.

Because the shell seeds Revit context objects, you can run small Revit API inspection scripts directly, for example:

print(doc.Title)

Status Colors

The status bar text updates after actions and briefly changes background color for feedback.

PaleGreen: save succeeded, for messages beginning with Saved.
LightGoldenrodYellow: info actions such as open, new file, run completed, and clear output.
MistyRose: errors and failed runs. Error color stays until the next status change.

Non-error informational colors fade back to transparent automatically after a short delay.

Tips

  • Use the shell for small inspection scripts, quick data checks, or API experiments.
  • If a run fails, read the red traceback in the output pane first; the status bar will also turn red.
  • Use CtrlD and CtrlL to work quickly on repeated line edits.
  • If you want to preserve your editor buffer before experimenting, save it first so the status bar confirms with PaleGreen.
  • Because this tool is modal, close it before continuing normal Revit commands.
